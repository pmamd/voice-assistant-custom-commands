# Test Infrastructure Documentation

**IMPORTANT:** This document describes the existing audio test infrastructure that MUST be used for voice assistant testing.

## Overview

The test suite has a complete **AUTOMATED** audio pipeline for end-to-end testing:

```
Text Input → Piper TTS → Audio File → Whisper STT → Verification
```

**This is FULLY AUTOMATED - NO MANUAL SPEAKING REQUIRED.**

All audio is synthetically generated using Piper TTS and verified using Whisper STT.
This validates that the audio pipeline (generation → transcription) works correctly.

For live microphone testing, you must run talk-llama separately and speak naturally.

## Key Components

### 1. AudioGenerator (`audio_generator.py`)

**Purpose:** Generate synthetic speech audio from text using Piper TTS

**Location:** `tests/audio_generator.py`

**Usage:**
```python
from audio_generator import AudioGenerator

generator = AudioGenerator(
    piper_bin="/home/paul/.local/bin/piper",
    model_dir="./external/piper-voices",
    output_dir="./tests/audio/inputs"
)

# Generate audio file
wav_file = generator.generate("stop", output_name="test_stop")
# Returns: Path to test_stop.wav
```

**Key Features:**
- Generates WAV files (22.05kHz, 16-bit, mono)
- Resamples to 16kHz for Whisper compatibility
- Adds 200ms leading silence to prevent word cut-off
- Pads to minimum 1500ms duration
- Uses same Piper voices as production
- Batch generation supported
- Consistent, reproducible audio (FULLY AUTOMATED)

**When to Use:**
- ✅ Automated testing of voice commands (NO manual speaking needed)
- ✅ Generating test fixtures for CI/CD
- ✅ Reproducing specific audio conditions
- ✅ Testing fast path keyword matching
- ✅ Testing LLM tool calling with audio
- ✅ Verifying Whisper transcription accuracy

---

### 2. AudioVerifier (`audio_verifier.py`)

**Purpose:** Verify TTS output by transcribing with Whisper STT

**Location:** `tests/audio_verifier.py`

**Usage:**
```python
from audio_verifier import AudioVerifier

verifier = AudioVerifier(
    whisper_bin="./build/bin/main",
    model_path="./whisper.cpp/models/ggml-tiny.en.bin"
)

# Verify with fuzzy matching
passed, similarity, text, confidence = verifier.verify_fuzzy(
    wav_file="output.wav",
    expected_text="Hello! How can I help you?",
    threshold=0.85
)

# Verify keywords
passed, matched, text, confidence = verifier.verify_keywords(
    wav_file="output.wav",
    keywords=["hello", "help"]
)
```

**Key Features:**
- Automatic audio resampling (22kHz → 16kHz)
- Multiple verification methods:
  - Exact text matching
  - Fuzzy matching (configurable threshold)
  - Keyword presence
- Confidence scores from Whisper
- Comprehensive error reporting

**When to Use:**
- ✅ Verifying TTS output quality
- ✅ Checking tool responses
- ✅ Validating conversation flow
- ✅ Testing audio pipeline end-to-end

**Test Types Supported:**

**When to Use:**
- ✅ Full pipeline testing

# 1. Generate test audio
